## Why

- The shell wrapper was a maintenance surface with no functional value: it existed solely to resolve the repo root and hand off to the Python script it wrapped. Because the central install is already the authoritative location for all shared-repo-memory helper scripts (per ADR-0002), routing git hooks through a repo-local shell file added fragility — the repo had to contain the wrapper for hooks to succeed — without adding any capability. Direct invocation removes this unnecessary coupling.

## What changed

- The catch-up rebuild hooks no longer depend on a repo-local shell script. The bootstrap script now generates post-checkout, post-merge, and post-rewrite hooks that call the central install's Python helper directly, using the same invocation pattern already established for other hook-driven helpers. The system version advances to reflect the hook architecture change. The design documentation is updated to describe the current invocation path.

## Evidence

- Design doc updated to reflect the new hook invocation path. ADR-0002 (one install, many symlinks) directly supports this pattern. The central install was already clean — no reference to the removed wrapper existed in the installer, confirming there was no secondary dependency to resolve.

## Next

- Repos bootstrapped before this change retain the old hook wiring pointing to the now-deleted wrapper; the idempotent repair in bootstrap-repo.py will regenerate correct hooks on the next session open. Any test fixtures or documentation that reference the wrapper path by name should be updated.
