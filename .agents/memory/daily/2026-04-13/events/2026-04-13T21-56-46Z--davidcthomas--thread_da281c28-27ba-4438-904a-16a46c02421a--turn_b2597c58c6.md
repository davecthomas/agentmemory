## Why

- The previous logging schema used 'agent' and 'provider-version' fields that implied every agentmemory caller is an AI agent. In practice, git hooks, the installer, and manual CLI invocations are not agents and were logging as 'unknown', making hook-trace records ambiguous when debugging mixed-runtime workflows. A runtime-neutral schema lets every caller self-identify accurately.

## What changed

- common.py: renamed the module-level log context state and the public API (detect_runtime_agent_id → detect_runtime_id, runtime_provider_version → runtime_version) while retaining the old names as backward-compatible aliases; changed the canonical log prefix format from [agent=...][provider-version=...] to [runtime=...][runtime-version=...]; introduced a _SCRIPT_RUNTIME_DEFAULTS map so non-agent scripts (bootstrap-repo.py, install.py, etc.) resolve to meaningful identifiers instead of falling through to the generic 'unknown' sentinel; changed the 'unknown' final fallback to 'system'. bootstrap-repo.py: set_runtime_log_context('bootstrap', 'n/a') now called at main() entry; generated post-checkout, post-merge, and post-rewrite hook scripts export AGENTMEMORY_RUNTIME_ID and AGENTMEMORY_RUNTIME_VERSION before invoking build-catchup.py, and now call build-catchup.py from the central install path rather than the repo-local run-catchup.sh. run-catchup.sh: exports AGENTMEMORY_RUNTIME_ID and AGENTMEMORY_RUNTIME_VERSION with 'git-hook'/'n/a' defaults.

## Evidence
